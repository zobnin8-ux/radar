# ДОП-ТЗ: концепция и БАЛАНС контента «Китайские штучки»

Дополнение к основному ТЗ. Базовый конвейер реализован — здесь **содержательная настройка отбора и баланса**. Это источник правды по ВКУСУ канала.

## Главный вопрос (финальная проверка в промпте)
При виде товара человек должен подумать хотя бы одно: «о, прикольно» / «что это вообще?» / «такого не видел» / «надо другу показать» / «забавно придумали».
**Тест вечеринки:** «Если показать товар человеку на вечеринке — найдётся ли повод рассказать о нём хотя бы одну фразу?» Нет — отклонить.

## 7 категорий и их доля в ленте

| # | Категория | Суть / примеры | Доля |
|---|-----------|----------------|------|
| 1 | **Гаджеты будущего** | очки-переводчики, VR/AR, умные кольца, мини-проекторы, электронные блокноты, носимое, необычные дисплеи | **≤ 20–25%** |
| 2 | **Неожиданные изобретения** ⭐ САМАЯ ВАЖНАЯ | «как до этого додумались?»: необычные кухонные/инструменты, нестандартные решения бытовых проблем | максимум |
| 3 | **Полезные штучки** (не скучные) | детектор скрытых камер, необычные фонарики, дорожные/аварийные гаджеты | — |
| 4 | **Мини-техника** | мини-принтеры, мини-пылесосы, мини-граверы, мини-лазеры, карманные измерители | — |
| 5 | **Для авто** (только необычное) | интересные инструменты, устройства безопасности, нестандартные решения. НЕ держатели/зарядки/коврики | — |
| 6 | **Для путешествий** (только интересное) | необычные органайзеры, дорожные гаджеты, переводчики, компактные решения | — |
| 7 | **Странные китайские штучки** | «сам бы не искал, но хочу посмотреть» — сложно объяснить рационально | **≤ 10–15%** |

> ⭐ **Категория 2 — приоритет №1.** Удивление от смекалки важнее тех-глянца. Дешёвое неожиданное изобретение должно обходить дорогой, но банальный VR-шлем. VR/AR — это лишь часть Кат.1 (≤25%), НЕ ядро канала.

## Маппинг на код-категории (`Category` enum)
Концепт-категории → 9 хэштег-категорий:
- Кат.1 → `future-stuff` · Кат.7 → `weird` · Кат.4 мини-техника → `gadgets`/`workshop` · Кат.5 → `auto` · Кат.6 → `travel` · Кат.2/3 → `gadgets`/`workshop`/`smart-home`/`edc` по смыслу.
Жёсткие доли: **`future-stuff` ≤ 25%**, **`weird` ≤ 15%**.

## Категорически НЕ публиковать
кабели, переходники, обычные карабины, обычные сумки, обычные органайзеры, чехлы, ремешки, контейнеры, коробки, крючки, расходники, обычные лампочки, обычные датчики, обычные розетки.
**+ аксессуары К устройству** (ремешки/линзы/чехлы/подставки/контроллеры к VR и пр.) — это НЕ сам гаджет.
Причина: решают проблему, но не создают интерес.

## ЧТО ИЗМЕНИТЬ В КОДЕ (дельта)

### Δ1 — Ключевые слова: широкий БАЛАНС тем (`src/sources/aliexpress.ts`)
Проблема прошлой версии: список был перекошен в VR (`smart glasses`+`AR glasses`+`VR headset`) → лента VR. Сделать набор **по темам, VR = одна строка**. Пример:
```ts
export const WOW_KEYWORDS = [
  // Кат.2 неожиданные изобретения (приоритет — больше запросов)
  "unusual kitchen gadget", "clever tool", "problem solving gadget",
  "genius invention gadget", "creative gadget",
  // Кат.4 мини-техника
  "mini printer", "mini vacuum", "mini engraver", "mini laser", "pocket gadget",
  // Кат.3 полезные с фишкой
  "hidden camera detector", "unusual flashlight", "emergency gadget", "endoscope camera",
  // Кат.1 будущее (VR — ОДНА строка, не доминирует)
  "smart ring", "mini projector", "translator device", "e-ink gadget", "VR AR gadget",
  // аудио/свет/робот
  "bone conduction headphones", "hologram display", "mini robot",
  // Кат.5/6 авто/тревел (необычное)
  "car safety gadget", "travel gadget",
];
export const WEIRD_KEYWORDS = ["weird gadget", "unusual gadget", "you didn't know you needed"]; // Кат.7, мало
export const GADGET_KEYWORDS = ["smart home gadget", "EDC tool", "desk gadget"]; // практичный трек, умеренно
```
Цель — чтобы пул был РАЗНЫМ по темам, а не моно-VR. Цена не важна (дешёвое ок).

### Δ2 — Промпт отбора (`src/ai/analyzeFinds.ts`)
Добавить/усилить:
- **Тест вечеринки** (см. выше) как финальный гейт.
- **Приоритет Кат.2:** высший `wow/curiosity` (8–10) за «как до этого додумались / нестандартное решение», даже если дёшево и просто. Это важнее тех-глянца.
- **Аксессуары К устройству** (ремешки/линзы/чехлы/контроллеры/подставки) → занижать как бытовуху (wow/curiosity 0–3) или `isPhysicalProduct=false`. (Чинит «VR + аксессуары к VR».)
- Сохранить эмоция-гейт, BORING_FILTER, banned-список, отказ транспорта/одежды/цены>$400.

### Δ3 — БАЛАНС при публикации (`src/utils/queuePick.ts` / `runBatchPublish.ts`) — ГЛАВНОЕ
Структурная защита от доминирования любой темы. При наборе батча (4 поста) из очереди (сортированной по `finalScore`):
- **`future-stuff` ≤ 1** на батч (≈≤25%).
- **`weird` ≤ 1** на батч (≈≤15%, кап уже есть — оставить).
- **Любая категория ≤ 2** на батч — не давать 4 однотипных.
- Если кандидат превышает кап своей категории — пропустить, взять следующий по `finalScore`.
Так лента всегда разнообразна, даже если пул перекошен.

## Критерии приёмки
- [ ] Батч из 4 = 4 РАЗНЫХ типа (не 4 VR, не 4 приколы, не 4 датчика).
- [ ] `future-stuff` ≤ ~1/батч, `weird` ≤ ~1/батч; остальное — разные темы.
- [ ] Видны неожиданные изобретения «как додумались» (Кат.2) — их много.
- [ ] Нет аксессуаров к VR, кабелей, обычных карабинов/сумок/датчиков.
- [ ] VR/AR появляется ИНОГДА, а не каждый пост.
- [ ] `npm run build` зелёный.

## Калибровка
- Мало разнообразия / лезет одна тема → проверить Δ3 (кап категорий) и расширить темы в `WOW_KEYWORDS`.
- Мало «неожиданных изобретений» → добавить запросов Кат.2 (`as seen on tiktok kitchen`, `clever home gadget`, `satisfying tool`) и поднять им приоритет в промпте.
- Лезут приколы вне #Weird → усилить «аксессуары/гэг → weird» в промпте.
